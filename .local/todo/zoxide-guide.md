# 建议

## zoxide 作为推荐项，不作为 Web 配置核心功能

### 结论

`zoxide` 更适合作为推荐工具写进文档或帮助说明，不建议作为 `omo` Web UI 第一版的核心配置能力。

### 原因

- `zoxide` 本身是独立工具
- 它的核心价值在于自动记忆目录，而不是让用户频繁手动配置
- 它的历史记录、模糊匹配和排序逻辑都由工具自身维护
- Web UI 很难为它提供真正高价值的“管理界面”

### 它更像什么

更像是：

- 外部工具推荐
- 命令行效率增强工具
- 文档中的配套建议

而不是：

- 需要在 `omo` Web UI 中重点管理的配置项

## 推荐放置位置

可以考虑放在这些位置：

- Web UI 的帮助说明
- README / 使用文档
- “效率工具推荐”区域
- PowerShell 配置说明页中的补充建议

## 推荐说明文案方向

可以简要告诉用户：

- `zoxide` 用于快速跳转常用目录
- 安装并初始化后会自动记忆目录历史
- 适合配合 `opencode` 一起使用

示例：

```powershell
z ezcoder
opencode .
```

## 产品取舍建议

第一版 Web 配置功能建议聚焦：

- `Set-Alias`
- `function`

不建议第一版投入到：

- `zoxide` 配置管理
- `zoxide` 历史记录管理
- `zoxide` 排名逻辑可视化

### 推荐原因

`alias` / `function`：

- 用户明确有配置需求
- 适合表单化
- 适合写入 `$PROFILE`
- 可视化管理价值高

`zoxide`：

- 基本不需要用户频繁配置
- 安装后主要靠自动学习
- Web 管理价值较低

## 后续可做但不优先

如果后面要补，也更适合做成轻量能力：

- 帮助卡片
- 安装提示
- 初始化命令展示
- 与 `opencode` 的搭配示例

而不是完整配置模块。
